# IDE 事件声音自定义插件 - 完整需求文档
## 一、项目概述
### 1. 项目名称
IDE 事件声音自定义插件（IDE Event Sounds）

### 2. 项目愿景
打造一款**跨 IDE 通用**的沉浸式开发声音提示工具，为 IntelliJ 平台（IDEA、PyCharm、WebStorm 等全系列）、后续 VS Code 提供事件声音自定义能力，让开发者通过专属语音/音效感知 IDE 关键事件，提升开发效率与沉浸感。

### 3. 核心定位
- 轻量级无侵入：不影响 IDE 性能、不修改源码，仅监听事件播放声音
- 高自定义：事件绑定、声音文件、消息匹配全支持用户配置
- 跨平台兼容：Windows/Mac/Linux 全系统支持
- 开箱即用：内置常用事件预设声音，无需复杂配置

## 二、核心功能总览
| 功能模块      | 核心能力                                                    |
| ------------- | ----------------------------------------------------------- |
| 事件-声音绑定 | 单个事件绑定唯一声音，支持解绑/替换                         |
| 预设声音库    | 内置开发高频事件语音/音效（如构建成功、编译失败、运行启动） |
| 自定义事件    | 支持用户手动添加 IDE 原生事件 Key，扩展监听范围             |
| 正则匹配      | 支持对事件消息内容做正则匹配，精准适配细分场景              |
| 配置管理      | 本地配置文件持久化，支持导入/导出配置                       |

## 三、版本迭代规划（分阶段落地）
### 0.0.1 版本（基础可用版）
#### 目标
实现核心闭环：**常用事件监听 + 预设声音绑定 + 基础播放**
#### 功能需求
1. **支持 IDE 范围**
   全 IntelliJ 平台 IDE（IDEA 2021.1 及以上版本兼容）
2. **预设事件列表（核心高频）**
   - 项目构建成功（Build Successful）
   - 项目构建失败（Build Failed）
   - 项目运行启动（Application Run）
   - 项目运行终止（Application Stop）
   - 代码编译完成（Compile Finished）
   - 测试用例执行成功（Test Passed）
   - 测试用例执行失败（Test Failed）
3. **声音配置**
   - 内置预设声音文件（女神语音：「应用构建完毕」「构建失败，请检查代码」等）
   - 采用**本地配置文件**（JSON/YAML）管理事件与声音映射关系
   - 声音格式：支持 MP3/WAV 通用格式
4. **基础能力**
   - 插件启动自动加载配置
   - 触发对应事件时**自动播放绑定声音**
   - 无界面、无干扰，后台静默运行

### 0.0.2 版本（自定义扩展版）
#### 目标
开放自定义能力，让用户绑定更多 IDE 原生事件
#### 功能需求

   - 支持用户**手动添加事件 Key**，绑定自定义声音
   - 提供 IntelliJ IDE 内**事件 Key 查询指南**（告诉用户在哪里查看可用事件）
2. **配置增强**
   - 配置文件支持新增/编辑/删除事件绑定
   - 支持用户替换预设声音为本地自定义音频文件
3. **兼容优化**
   - 兼容更多小众 IntelliJ 原生事件
   - 解决不同系统声音播放兼容性问题

### 0.0.3 版本（高级匹配版）

实现消息正则匹配，覆盖更细分、更个性化的场景
#### 功能需求
1. **正则匹配核心功能**
   - 支持为事件配置**消息内容正则表达式**
   - 只有事件消息匹配正则时，才播放对应声音
2. **场景示例**
   - 构建事件：匹配「成功」播放成功音，匹配「错误」播放失败音
   - 测试事件：匹配「全部通过」「部分失败」区分提示
3. **配置升级**
   - 配置文件新增 `regex` 字段，支持编写正则规则
   - 提供正则语法示例，降低用户使用门槛
4. **体验优化**
   - 支持测试正则匹配效果
   - 支持音量调节、播放优先级设置（避免多声音重叠）

## 四、详细功能需求
### 1. 事件管理模块
1.1 内置事件
- 固定支持 0.0.1 版本定义的 7 个高频事件
- 事件不可删除，仅可替换声音
1.2 自定义事件
- 用户通过**事件 Key** 添加 IDE 原生事件
- 支持查看已添加的自定义事件列表
- 支持编辑事件绑定的声音、删除自定义事件
1.3 事件监听规则
- 只监听 IDE 全局标准事件，不监听项目私有事件
- 事件触发后**立即异步播放声音**，不阻塞 IDE 操作

### 2. 声音管理模块
2.1 声音来源
- 内置预设声音（插件打包自带，无需用户下载）
- 用户自定义本地声音（MP3/WAV 格式）
2.2 声音播放规则
- 单次事件仅播放**一个声音**，不重复播放
- 支持快速切换开关：启用/禁用所有声音提示
2.3 格式要求
- 音频时长：建议 1-3 秒（避免过长干扰开发）
- 格式：MP3、WAV（全平台兼容）

### 3. 配置文件规范（核心）
#### 配置文件格式：`config.json`
```json
{
  "version": "0.0.1",          // 配置版本
  "enable": true,              // 全局开关
  "sounds": [
    {
      "eventKey": "build.success",  // 事件唯一标识（必填）
      "soundPath": "preset/build_success.mp3", // 声音路径
      "name": "构建成功",        // 事件名称（展示用）
      "regex": ""               // 正则匹配（0.0.3 版本启用）
    },
    {
      "eventKey": "build.failed",
      "soundPath": "preset/build_failed.mp3",
      "name": "构建失败",
      "regex": ""
    }
  ]
}
```
#### 配置文件路径
- Windows：`C:\Users\用户名\.ide-event-sounds\config.json`
- Mac：`~/.ide-event-sounds/config.json`
- Linux：`~/.ide-event-sounds/config.json`

### 4. 正则匹配规则（0.0.3+）
- 基于 Java 标准正则语法（IntelliJ 平台）
- 空正则 = 不匹配，直接触发声音
- 正则匹配成功 = 播放声音，匹配失败 = 不播放
- 支持基础正则：`.*成功.*`、`^错误\d+.*` 等

## 五、非功能需求
### 1. 性能要求
- 插件内存占用 < 20MB
- 事件响应延迟 < 100ms
- 声音播放不卡顿、不影响 IDE 编译/运行速度

### 2. 兼容性
- IntelliJ：支持 2021.1 ~ 最新版
- 系统：Windows 10+、MacOS 11+、Linux Ubuntu 20.04+
- 声音：全系统兼容 MP3/WAV

### 3. 稳定性
- 插件异常不崩溃 IDE
- 配置文件错误自动恢复默认配置
- 声音文件丢失不报错，自动跳过播放

### 4. 易用性
- 零学习成本：安装即用，默认启用预设声音
- 配置简单：文本编辑器即可修改配置
- 无广告、无收集用户数据

## 六、用户场景示例
1. **日常开发**
   写完代码点击构建，耳机里响起「应用构建完毕」，无需看 IDE 即可继续开发

   运行测试用例，成功播放「测试全部通过」，失败播放「测试存在错误」
3. **自定义场景（0.0.3）**
   监听「运行日志事件」，正则匹配 `端口 \d+ 已启用`，播放服务启动成功音
4. **团队共享**
   导出配置文件给同事，统一团队开发提示音，提升协作沉浸感
